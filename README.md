# SRE-Job-Description
2026年SRE/DevOps/AI Infra学习路线，人肉爬取boss直聘信息提取

本想在 GitHub 找脚本，批量爬取 BOSS 直聘上 SRE、DevOps、运维、AI infra 相关岗位的 JD，结果发现现有脚本大多失效。

BOSS 直聘反爬严格，浏览器开发者工具、控制台无法打开，开启即被强制退出，甚至无法复制文本。


若要强行脚本抓取，只能用 AI 模拟人工操作或浏览器无头模式调试，但编码加调试耗时过久，性价比极低。


最终我选择手动截图，通过大模型 OCR 提取文字，再按关键词频率压缩、总结，整理出这份学习路径，分享供大家参考。

## 四、学习优先级建议
### 1. 通用基础（必学）
- 编程语言：Python + Shell（掌握基础语法、常用库、运维脚本编写）
- 云原生基础：Docker + Kubernetes核心概念（调度、Pod、Service、ConfigMap）
- 监控基础：Prometheus + Grafana（指标采集、仪表盘、告警规则）

### 2. 方向聚焦（按需选择）
- **AI Infra方向**：GPU（CUDA）+ 大模型推理（vLLM/KV Cache）+ K8s GPU调度
- **SRE方向**：可观测性（ELK/Jaeger）+ 混沌工程 + SLO/SLA管理
- **云原生方向**：K8s Operator + 容器网络（eBPF/Cilium）+ 多集群联邦
- **大数据SRE方向**：Spark/Flink + HDFS + Kafka + 性能调优
- **中间件SRE方向**：Redis/Kafka/ES集群运维 + 高可用架构设计

### 3. 进阶提升（加分项）
- 底层优化：eBPF/DPDK、CUDA算子开发、低比特量化
- 云厂商生态：阿里云SLS/腾讯云VPC/华为云APM
- 认证加持：CKA/CKS、AWS DevOps、SRE相关认证
